# 情趣内衣尺码图技能

## 0. 使用场景

当任务是根据第一批情趣内衣 / 睡衣 Excel 尺码数据，批量生成俄语 3:4 竖版 WB/Ozon 商品尺码图 PNG 时，必须使用本技能。

本技能不是固定套模板。每张图都必须由以下部分动态生成：

**模板版式 + 当前货号实际字段 + 当前货号实际尺寸 + 当前商品图款式校验 + 当前画面空白布局**

宁愿少显示一个经过验证的字段，也不要显示原始表里不存在或无法正确画线的字段。

---

## 1. 最高优先级规则

1. 不要直接修改旧 PNG；必须修改最新生成代码 / 最新模板逻辑，然后重新渲染新 PNG。
2. 模板图只参考版式、色彩、线稿风格，不参考任何示例数字。
3. Excel 是尺寸数值来源；商品图只用于款式确认、模板校验、线稿校验，不用于估算尺寸。
4. 每张图必须根据当前货号自己的原始尺寸动态生成。
5. 两个表格都必须使用“尺码横排”。
6. A/B/C/D/E/F 是纵向测量项目行，不再作为横向列。
7. 没有数据的字段直接删除，不要显示整行 `—`。
8. 图例、测量线、右下表、底部提示必须完全一致。
9. 图例注释不能只固定在左下角；必须允许使用右侧表格下方和页面中下部空白。
10. 字段多时，不允许缩小图例文字硬塞；优先调整布局、利用空白、两列/三列排版、删除低优先级字段。
11. 先只生成 5 张样图：`5346`、`7889`、`968`、`7171`、`7975`。
12. 样图输出到新文件夹 `size_chart_output_v2_sample`，不要覆盖旧版本。
13. 样图确认后才允许跑完整第一批。

## 2. 输入文件

必须读取：

1. `GPT用_原始尺码数据总表_第一批_已补身高体重依据_作图选码版.xlsx`
   - `GPT总表`
   - `原始尺寸长表`

2. `第一批_货号模板对应关系.xlsx`
   - 用于判断每个 `货号/款号` 使用哪个模板。

3. 当前货号对应的产品图片 / 产品图片文件
   - 用于确认商品款式、模板是否合理、线稿是否匹配。
   - 不允许从商品图估算尺寸。

4. 7 张模板图
   - 只参考视觉风格、线稿类型和大致排版。
   - 禁止复制模板图中的任何示例数字。

## 3. 图片整体规格

- 比例：3:4 竖版
- 推荐画布：900×1200 或等比例高清 PNG
- 输出格式：PNG
- 风格：简洁、干净、不拥挤，适合 Wildberries / Ozon 商品附图
- 背景：白色
- 标题栏 / 表头：浅粉色
- 文字：黑色或深灰色俄语
- 表格线：细灰线
- 左侧：服装线稿 + 粉色虚线测量线
- 右侧：两个横排尺码表
- 底部：两行短提示

禁止写：

- `Для рынка РФ`
- `ориентировочные данные первой партии`
- 冗长解释段落
- 与当前字段无关的提示语

## 4. 商品图识别与线稿校验

生成每张尺码图前，必须查看当前货号对应的产品图片 / 产品图片文件。

商品图的作用不是读取尺寸数值，而是用于确认：商品款式、模板是否合理、线稿是否匹配、测量线是否能画在正确结构上。

尺寸数值只能来自 Excel 表格，不能从商品图估算。

生成前必须根据商品图确认：

1. 商品是连体衣、吊带裙、睡衣套装、长裙、短裙还是其他款式。
2. 商品是否为分体套装。
3. 商品是否包含短裤、长裤、内裤或其他下装。
4. 商品是否有长袖 / 短袖 / 无袖 / 吊带。
5. 商品长度是短款、中长款还是长款。
6. 当前模板线稿是否与商品图一致。
7. 测量线是否画在正确商品结构上。

如果商品图与模板关系表冲突：

1. 不要强行生成最终图。
2. 在检查表中标记 `商品图与模板冲突`。
3. 写明冲突原因。
4. 输出到人工确认清单。

冲突示例：

- 模板表写 T3 普通短裤睡衣，但商品图有长袖，应标记为 T3A 或人工确认。
- 模板表写吊带裙，但商品图显示分体短裤套装，不能用吊带裙线稿。
- 模板表写长款吊带裙，但商品图明显是短款，需要人工确认。
- 尺寸字段包含短裤腰围，但商品图没有短裤/内裤线稿，不能把短裤腰围画到裙身上。
- 商品图是睡衣上衣，但线稿画成正式衬衫、外套、职业装，线稿不合格，需要重画。

线稿必须符合商品图基本结构：

- 连体衣必须像连体衣，不要画成普通吊带裙。
- 吊带裙必须像吊带睡裙，不要画成正式连衣裙。
- 睡衣套装必须像家居睡衣，不要画成衬衫、西装、外套。
- 长袖短裤睡衣必须画成长袖睡衣上衣 + 短裤，不要画成纽扣衬衫、开衫或职业衬衫。
- 长裤睡衣必须显示上衣 + 长裤。
- 短裤睡衣必须显示上衣 + 短裤。

如果无法从商品图确认款式，则以模板关系表为主，但必须在检查表中标记：

`商品图未确认，仅按模板关系生成`

## 5. 表格布局：尺码横排

### 5.1 右上表：推荐参数

标题：

`Рекомендуемые параметры`

格式：

| Параметр | S | M | L | XL | XXL |
|---|---|---|---|---|---|
| Рост, см | ... | ... | ... | ... | ... |
| Вес, кг | ... | ... | ... | ... | ... |

规则：

- `Рост, см` 来源：`GPT总表` 的 `推荐身高（俄市场预估）`。
- `Вес, кг` 来源：`GPT总表` 的 `推荐体重（俄市场预估）`。
- 单元格里不要重复写 `см` / `кг`，单位只写在行名里。
- 尺码列只显示该货号实际存在的尺码。
- 没有 XXL 就不要显示 XXL。
- 只有 S 码就只显示 S。

### 5.2 右下表：商品实测尺寸

标题：

`Замеры изделия`

格式：

| Замер | S | M | L | XL | XXL |
|---|---|---|---|---|---|
| A | ... | ... | ... | ... | ... |
| B | ... | ... | ... | ... | ... |
| C | ... | ... | ... | ... | ... |

规则：

- 数据来源：`原始尺寸长表`。
- 必须按 `批次 = 第一批`、`货号/款号`、`尺码`、`中文部位`、`原始尺寸数值` 逐项读取。
- 不允许使用模板图里的示例数值。
- 不允许把同类多个货号的数据合并成范围。
- 不允许编造缺失尺寸。
- A/B/C/D/E/F 作为纵向行显示。
- 测量部位越多，越应该纵向增加行，不要横向挤表格。
- 如果某个测量部位在当前货号所有尺码中都没有数据，删除该字段，不要显示整行 `—`。

## 6. 模板分类

模板编号以 `第一批_货号模板对应关系.xlsx` 为准，但必须用商品图做二次校验。

### T1：连体衣 / Боди

标题：

`Боди / комбинезон`

默认优先字段：

1. 衣长 / 总长 / 衣长不含肩带
2. 胸围 / 下胸围
3. 腰围
4. 臀围
5. 肩带长

俄语字段：

- 衣长不含肩带 → `длина без бретелей`
- 衣长 / 总长 → `длина изделия`
- 胸围 → `грудь`
- 下胸围 → `под грудью`
- 腰围 → `талия`
- 臀围 → `бёдра`
- 肩带长 → `длина бретели`

规则：

- 如果没有臀围，不显示 `бёдра`。
- 如果有肩带长，可以显示 `длина бретели`。
- 如果显示肩带长，测量线必须画在肩带位置。

### T2：分体长裤睡衣

标题：

`Пижамный комплект с брюками`

默认优先字段：

1. 上衣长
2. 胸围 / 下胸围
3. 裤腰围
4. 臀围
5. 裤长
6. 裤内长

俄语字段：

- 上衣长 → `длина верха`
- 胸围 → `грудь`
- 下胸围 → `под грудью`
- 裤腰围 → `талия брюк`
- 臀围 → `бёдра`
- 裤长 → `длина брюк`
- 裤内长 → `внутренний шов` 或 `длина по внутреннему шву`

注：图例空间有限时，优先使用 `внутренний шов`，避免长句压小字体。

### T3：分体短裤睡衣

标题：

`Пижамный комплект с шортами`

默认优先字段：

1. 上衣长
2. 胸围 / 下胸围
3. 短裤腰围
4. 臀围
5. 短裤长 / 裤侧长
6. 肩带长，如有

俄语字段：

- 上衣长 → `длина верха`
- 胸围 → `грудь`
- 下胸围 → `под грудью`
- 短裤腰围 → `талия шорт`
- 臀围 → `бёдра`
- 短裤长 → `длина шорт`
- 肩带长 → `длина бретели`

### T3A：长袖短裤睡衣，7889 专用

标题必须是：

`Пижамный комплект с шортами и длинным рукавом`

默认优先字段：

1. 上衣长
2. 下胸围 / 胸围
3. 袖长
4. 短裤腰围
5. 臀围
6. 短裤长 / 裤侧长

俄语字段：

- 上衣长 → `длина верха`
- 下胸围 → `под грудью`
- 胸围 → `грудь`
- 袖长 → `длина рукава`
- 短裤腰围 → `талия шорт`
- 臀围 → `бёдра`
- 短裤长 → `длина шорт`

如果当前货号包含袖长字段，必须使用 T3A 或带袖线稿，不能套普通 T3。

### T4：短款吊带裙

标题：

`Короткая сорочка на бретелях`

默认优先字段：

1. 衣长 / 裙长
2. 胸围
3. 下胸围
4. 腰围
5. 肩带长

俄语字段：

- 杯顶至下摆 / 不含肩带 → `длина без бретелей`
- 肩顶至下摆 / 含肩带 → `длина с бретелями`
- 裙长 / 衣长泛称 → `длина сорочки`
- 胸围 → `грудь`
- 下胸围 → `под грудью`
- 腰围 → `талия`
- 肩带长 → `длина бретели`

### T5：常规吊带睡裙

标题：

`Сорочка на бретелях средней длины`

默认优先字段：

1. 衣长 / 裙长
2. 胸围
3. 下胸围
4. 腰围
5. 臀围，如有
6. 肩带长，如有

俄语字段同 T4。

特别注意：

如果原字段是 `内裤腰围`、`裤腰围`、`腰围（腰顶全围）`，不要直接当成裙身 `талия`。如果要展示内裤腰围，必须画单独内裤/下装线稿并写 `талия трусиков`；否则删除该字段。

### T6：长款吊带裙 / Макси

标题：

`Длинная сорочка / макси`

默认优先字段：

1. 衣长 / 裙长
2. 胸围
3. 腰围
4. 臀围
5. 下胸围
6. 肩带长

俄语字段：

- 长款裙长 / 衣长 → `длина сорочки`
- 不含肩带 → `длина без бретелей`
- 含肩带 → `длина с бретелями`
- 胸围 → `грудь`
- 下胸围 → `под грудью`
- 腰围 → `талия`
- 臀围 → `бёдра`
- 肩带长 → `длина бретели`

## 7. 动态字段与 A/B/C/D/E/F 分配

A/B/C/D/E/F 不固定代表某个部位。必须先分析当前货号的 `available_fields`，再决定最终保留字段，最后重新分配字母。

建议排序：

1. 长度类
2. 胸围 / 下胸围
3. 腰围
4. 臀围
5. 肩带 / 袖长 / 裤长 / 短裤长等结构尺寸

示例：5346

原始字段：

- 衣长不含肩带：55 / 56 / 57 / 58 / 59
- 胸围：76 / 80 / 84 / 90 / 94
- 腰围：64 / 68 / 72 / 78 / 82
- 肩带长：36 / 36 / 36 / 36 / 36

正确输出：

- A — `длина без бретелей`
- B — `грудь`
- C — `талия`
- D — `длина бретели`

右下表：

| Замер | S | M | L | XL | XXL |
|---|---|---|---|---|---|
| A | 55 | 56 | 57 | 58 | 59 |
| B | 76 | 80 | 84 | 90 | 94 |
| C | 64 | 68 | 72 | 78 | 82 |
| D | 36 | 36 | 36 | 36 | 36 |

错误输出：

- D — `бёдра`

因为 5346 原表没有臀围。

## 8. 测量线位置规则

测量线必须与字段含义一致：

- `длина изделия` / `длина сорочки`：整体纵向长度
- `длина без бретелей`：从杯顶 / 上边缘到下摆，不含肩带
- `длина с бретелями`：从肩带顶点到下摆，含肩带
- `грудь`：胸部横向线
- `под грудью`：杯底 / 下胸围横向线
- `талия`：裙身腰部位置
- `талия брюк`：裤子腰头位置
- `талия шорт`：短裤腰头位置
- `талия трусиков`：内裤腰头位置
- `бёдра`：臀部位置
- `длина бретели`：肩带位置
- `длина рукава`：袖子外侧
- `длина брюк`：裤子外侧纵向
- `длина шорт`：短裤侧边纵向
- `внутренний шов` / `длина по внутреннему шву`：裤内侧缝线位置

如果无法准确画出对应位置，则不要显示该字段。

---

## 9. 数值处理规则

1. 商品实测尺寸必须来自当前货号自己的 `原始尺寸长表`。
2. 同一部位同一尺码只有一个值时，直接写单值，例如 `55`。
3. 同一部位同一尺码存在平铺值和拉伸值时，才允许写范围，例如 `65–86`。
4. 不要把不同货号的数据合并成范围。
5. 不要自动把“平铺值”乘以 2。原表中很多“平铺”数值已经是全围，必须保留原值。
6. 所有商品实测尺寸单位默认 cm，表格中不用每格重复写 cm。
7. 小数保留原始值，例如 `64.5`、`67.5`、`42.6`。
8. 不要编造缺失数据。

---

## 10. 特殊字段处理

### 10.1 内裤 / 下装字段

如果字段名包含：

- `内裤腰围`
- `T裤腰围`
- `裤腰围`
- `腰顶全围`
- `裤长`
- `裤侧长`
- `裤内长`

这些属于下装字段，不要画在裙身上。

如果当前模板没有下装线稿，原则上不要显示这些字段。除非额外画出内裤 / 短裤线稿，并在图例中明确写：

- `талия трусиков`
- `талия шорт`
- `длина шорт`

### 10.2 肩带字段

如果显示肩带长：

- 图例写 `длина бретели`
- 测量线画在肩带上
- 不要画在衣身侧边

### 10.3 袖长字段

如果有袖长：

- 必须使用 T3A 或带袖线稿
- 图例写 `длина рукава`
- 测量线画在袖子上

### 10.4 长度字段

如果原字段写“不含肩带”，俄语必须写：

`длина без бретелей`

如果原字段写“含肩带 / 肩顶至下摆”，俄语必须写：

`длина с бретелями`

如果原字段没有明确含不含肩带，可写：

`длина сорочки` 或 `длина изделия`

## 11. 字体大小与手机端可读性

尺码图必须优先保证手机端可读性，不要为了塞更多字段把字缩得太小。

900×1200 PNG 参考字号：

1. 主标题：36–44 px
2. 表格标题：26–30 px
3. 表头文字：22–24 px
4. 表格数字：22–26 px
5. A/B/C/D/E/F 图例注释：24–28 px
6. 左侧测量线圆形字母：26–32 px
7. 底部提示：20–24 px

硬性规则：

- A/B/C/D/E/F 图例注释不能小于表格数字。
- 左侧测量线圆形字母必须比表格数字更醒目。
- 字段多时，不允许缩小图例文字硬塞。
- 字段多时优先使用两列/三列图例、缩小线稿、调整留白、利用右侧空白。
- 如果仍然拥挤，删除低优先级字段，而不是缩小字体。
- 图例文字在手机端必须清楚可读。

---

## 12. 图例注释区布局规则

A/B/C/D/E/F 图例注释不要死板限制在左侧线稿下面。

图例注释区可以根据空白区域动态放置，优先保证可读性。

允许使用以下区域：

1. 左侧线稿下方区域
2. 左侧线稿与右侧表格之间的空白区域
3. 右下表格下方、底部提示框上方的空白区域
4. 页面中下部横向空白区域

如果右侧表格下方有明显空白，必须优先利用这块空间放大图例注释，而不是把注释全部压在左下角。

字段少时：

- 图例可以放在线稿下方
- 单列或两列均可

字段多时：

- 图例可以横向展开到右侧空白区域
- 可以放在右下表格下方
- 可以使用两列或三列排版
- 每条图例之间必须有足够间距

硬性要求：

- 图例注释不能只固定在最左侧。
- 如果右下表格下方有空地，必须利用这块空地。
- 图例文字不能小于表格数字。
- 不允许把图例压缩成很小一团。
- 不允许为了保持左下角固定位置而牺牲可读性。
- 如果图例字段超过 4 个，禁止只放在左侧窄区域，必须允许图例横跨页面中下部，使用右侧表格下方空白。
- 图例区的目标是手机端清楚可读，不是全部挤在左边。

## 13. 底部提示

底部只保留两句。

第一句固定：

`Если вы между двумя размерами — выбирайте больший.`

第二句必须根据最终显示字段动态生成，不要提到图中没有显示的字段。

建议：

- 字段包含袖长：`Для комплекта важны длина рукава, талия шорт и бёдра. Допуск ±1–3 см.`
- T1 连体衣：`Для боди ориентируйтесь на рост и длину изделия. Допуск ±1–3 см.`
- T2 长裤睡衣：`Для комплекта важны талия, бёдра и длина брюк. Допуск ±1–3 см.`
- T3 短裤睡衣：`Для комплекта важны талия шорт, бёдра и посадка верха. Допуск ±1–3 см.`
- T4/T5 且显示 `грудь`、`талия`、`длина`：`Для сорочки важны грудь, талия и длина. Допуск ±1–3 см.`
- 只显示 `под грудью`、长度、肩带长：`Для этой модели важны длина изделия, обхват под грудью и длина бретели. Допуск ±1–3 см.`
- T6 长款：`Для длинной сорочки важны рост и длина изделия. Допуск ±1–3 см.`
- T6 如果显示腰围/臀围/下胸围/肩带长，可写：`Для длинной сорочки важны рост, длина изделия и посадка по фигуре. Допуск ±1–3 см.`

## 14. 线稿风格要求

线稿不仅要能画测量线，还要符合商品类型。

- 睡衣套装线稿必须像睡衣/家居服，不要画成正式衬衫、外套、职业装。
- 情趣内衣 / 吊带裙线稿要保持轻薄、贴身、柔和，不要画成普通正式连衣裙。
- 长袖短裤睡衣 T3A 的上衣应是长袖睡衣上衣，不要画成纽扣衬衫或开衫。
- 如果线稿风格和商品类型明显不符，即使表格数据正确，也判定为需要重画。

## 18. 问题处理

1. 完全缺尺码数据：不生成图片，记录到检查表。
2. 只有 S 码：只显示 S 列，不编造 M/L/XL。
3. 缺少某个部位：删除该字段，不显示整行 `—`。
4. 字段语义不确定：保留更核心字段，删除可疑字段。
5. 下装字段无法正确画线：删除字段，不要画在裙身上。
6. 推荐身高体重缺失：可以只生成 `Замеры изделия` 表，但检查表中标记推荐参数缺失。
7. 商品图与模板明显冲突：不要强行生成最终图，进入人工确认清单。
8. 线稿风格不符合商品图：重画线稿，不要仅因数据正确就通过。
9. 图例太小或挤在左下角：重排图例区，利用右侧表格下方和中下部空白。

## 19. 最终质量标准

每张图必须满足：

1. 右上表是横排尺码。
2. 右下表是横排尺码。
3. A/B/C/D/E/F 是纵向测量项目。
4. 图例 = 测量线 = 右下表行 = 底部提示。
5. 没有整行 `—`。
6. 没有原表不存在的部位。
7. 没有把多个货号合并成范围。
8. 没有把内裤/裤子尺寸画在裙身上。
9. 没有模板图里的旧示例数字。
10. 已查看商品图并完成款式/线稿校验。
11. 线稿风格符合商品图，不像错误类别服装。
12. 图例注释手机端可读。
13. 字段多时已利用右下表格下方和中下部空白，而不是全部挤在左下角。
14. 图片简洁，不密集，适合 Wildberries / Ozon 商品附图。

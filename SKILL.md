# 情趣内衣尺码图技能

## 使用场景

当任务是根据第一批情趣内衣 / 睡衣 Excel 尺码数据，批量生成俄语 3:4 竖版 WB/Ozon 商品尺码图 PNG 时，必须使用本技能。

本技能不是固定套模板。每张图都必须由以下三部分动态生成：

**模板版式 + 当前货号实际字段 + 当前货号实际数值**

宁愿少显示一个经过验证的字段，也不要显示原始表里不存在的字段。

## 输入文件

必须读取：

1. `GPT用_原始尺码数据总表_第一批_已补身高体重依据_作图选码版.xlsx`
   - `GPT总表`
   - `原始尺寸长表`

2. `第一批_货号模板对应关系.xlsx`
   - 用于判断每个 `货号/款号` 使用哪个模板。

3. 7 张模板图
   - 模板图只参考视觉风格、线稿类型和大致排版。
   - 禁止复制模板图中的任何示例数字。

## 最高优先级规则

1. 不要直接修改旧 PNG；必须修改最新生成逻辑，然后重新渲染新 PNG。
2. 模板图只参考版式，不参考任何示例数字。
3. 每张图必须根据当前货号自己的原始尺寸动态生成。
4. 两个表格都必须改为“尺码横排”。
5. A/B/C/D/E/F 是纵向测量项目行，不再作为横向列。
6. 没有数据的字段直接删除，不要显示整行 `—`。
7. 图例、测量线、右下表、底部提示必须完全一致。
8. 先只生成 5 张样图：`5346`、`7889`、`968`、`7171`、`7975`。
9. 样图输出到新文件夹 `size_chart_output_v2_sample`，不要覆盖旧版本。
10. 样图确认后才允许跑完整第一批。

## 图片规格

- 比例：3:4 竖版
- 格式：PNG
- 风格：简洁、干净、不拥挤，适合 Wildberries / Ozon 商品附图
- 背景：白色
- 标题栏 / 表头：浅粉色
- 文字：黑色或深灰色俄语
- 表格线：细灰线
- 左侧：服装线稿 + 粉色虚线测量线
- 右侧：两个横排尺码表
- 底部：两行短提示

禁止写：

- `Для рынка РФ`
- `ориентировочные данные первой партии`
- 冗长解释段落

## 表格布局：尺码横排

### 右上表：推荐参数

标题：

`Рекомендуемые параметры`

格式：

| Параметр | S | M | L | XL | XXL |
|---|---|---|---|---|---|
| Рост, см | ... | ... | ... | ... | ... |
| Вес, кг | ... | ... | ... | ... | ... |

规则：

- `Рост, см` 来源：`GPT总表` 的 `推荐身高（俄市场预估）`。
- `Вес, кг` 来源：`GPT总表` 的 `推荐体重（俄市场预估）`。
- 单元格里不要重复写 `см` / `кг`，单位只写在行名里。
- 尺码列只显示该货号实际存在的尺码。
- 没有 XXL 就不要显示 XXL。
- 只有 S 码就只显示 S。

### 右下表：商品实测尺寸

标题：

`Замеры изделия`

格式：

| Замер | S | M | L | XL | XXL |
|---|---|---|---|---|---|
| A | ... | ... | ... | ... | ... |
| B | ... | ... | ... | ... | ... |
| C | ... | ... | ... | ... | ... |

规则：

- 数据来源：`原始尺寸长表`。
- 必须按 `批次 = 第一批`、`货号/款号`、`尺码`、`中文部位`、`原始尺寸数值` 逐项读取。
- 不允许使用模板图里的示例数值。
- 不允许把同类多个货号的数据合并成范围。
- 不允许编造缺失尺寸。
- A/B/C/D/E/F 作为纵向行显示。
- 测量部位越多，越应该纵向增加行，不要横向挤表格。
- 如果某个测量部位在当前货号所有尺码中都没有数据，删除该字段，不要显示整行 `—`。

## 模板分类

模板编号以 `第一批_货号模板对应关系.xlsx` 为准。

### T1：连体衣 / Боди

标题：

`Боди / комбинезон`

默认优先字段：

1. 衣长 / 总长 / 衣长不含肩带
2. 胸围 / 下胸围
3. 腰围
4. 臀围
5. 肩带长

俄语字段：

- 衣长不含肩带 → `длина без бретелей`
- 衣长 / 总长 → `длина изделия`
- 胸围 → `грудь`
- 下胸围 → `под грудью`
- 腰围 → `талия`
- 臀围 → `бёдра`
- 肩带长 → `длина бретели`

规则：

- 如果没有臀围，不显示 `бёдра`。
- 如果有肩带长，可以显示 `длина бретели`。
- 如果显示肩带长，测量线必须画在肩带位置。

4. 臀围
5. 裤长
6. 裤内长

3. 短裤腰围
4. 臀围
5. 短裤长 / 裤侧长
6. 肩带长，如有

俄语字段：

- 上衣长 → `длина верха`
- 胸围 → `грудь`
- 下胸围 → `под грудью`
- 短裤腰围 → `талия шорт`

`Пижамный комплект с шортами и длинным рукавом`

默认优先字段：

1. 上衣长
2. 下胸围 / 胸围
3. 袖长
4. 短裤腰围
5. 臀围
6. 短裤长 / 裤侧长

俄语字段：

- 上衣长 → `длина верха`
- 下胸围 → `под грудью`
- 胸围 → `грудь`
- 袖长 → `длина рукава`
- 短裤腰围 → `талия шорт`
- 臀围 → `бёдра`

### T4：短款吊带裙

`Короткая сорочка на бретелях`

默认优先字段：

1. 衣长 / 裙长
2. 胸围
3. 下胸围
4. 腰围
5. 肩带长

俄语字段：

- 裙长 / 衣长泛称 → `длина сорочки`

### T5：常规吊带睡裙

标题：

`Сорочка на бретелях средней длины`

4. 腰围

6. 肩带长，如有

## 动态字段与 A/B/C/D/E/F 分配

A/B/C/D/E/F 不固定代表某个部位。必须先分析当前货号的 `available_fields`，再决定最终保留字段，最后重新分配字母。

建议排序：

1. 长度类
2. 胸围 / 下胸围
3. 腰围
4. 臀围
5. 肩带 / 袖长 / 裤长 / 短裤长等结构尺寸

示例：5346

原始字段：

- 衣长不含肩带：55 / 56 / 57 / 58 / 59
- 胸围：76 / 80 / 84 / 90 / 94
- 腰围：64 / 68 / 72 / 78 / 82
- 肩带长：36 / 36 / 36 / 36 / 36

正确输出：

- A — `длина без бретелей`
- B — `грудь`
- C — `талия`
- D — `длина бретели`

右下表：

| Замер | S | M | L | XL | XXL |
|---|---|---|---|---|---|
| A | 55 | 56 | 57 | 58 | 59 |
| B | 76 | 80 | 84 | 90 | 94 |
| C | 64 | 68 | 72 | 78 | 82 |
| D | 36 | 36 | 36 | 36 | 36 |

错误输出：

- D — `бёдра`

因为 5346 原表没有臀围。

## 测量线位置规则

测量线必须与字段含义一致：

- `длина изделия` / `длина сорочки`：整体纵向长度
- `длина без бретелей`：从杯顶 / 上边缘到下摆，不含肩带
- `длина с бретелями`：从肩带顶点到下摆，含肩带
- `грудь`：胸部横向线
- `под грудью`：杯底 / 下胸围横向线
- `талия`：裙身腰部位置
- `талия брюк`：裤子腰头位置
- `талия шорт`：短裤腰头位置
- `талия трусиков`：内裤腰头位置
- `бёдра`：臀部位置
- `длина бретели`：肩带位置
- `длина рукава`：袖子外侧
- `длина брюк`：裤子外侧纵向
- `длина шорт`：短裤侧边纵向
- `длина по внутреннему шву`：裤内侧缝线位置

如果无法准确画出对应位置，则不要显示该字段。

## 数值处理规则

1. 商品实测尺寸必须来自当前货号自己的 `原始尺寸长表`。
2. 同一部位同一尺码只有一个值时，直接写单值，例如 `55`。
3. 同一部位同一尺码存在平铺值和拉伸值时，才允许写范围，例如 `65–86`。
4. 不要把不同货号的数据合并成范围。
5. 不要自动把“平铺值”乘以 2。原表中很多“平铺”数值已经是全围，必须保留原值。
6. 所有商品实测尺寸单位默认 cm，表格中不用每格重复写 cm。
7. 小数保留原始值，例如 `64.5`、`67.5`、`42.6`。
8. 不要编造缺失数据。

## 特殊字段处理

### 内裤 / 下装字段

如果字段名包含：

- `内裤腰围`
- `T裤腰围`
- `裤腰围`
- `腰顶全围`
- `裤长`
- `裤侧长`
- `裤内长`

这些属于下装字段，不要画在裙身上。

如果当前模板没有下装线稿，原则上不要显示这些字段。除非额外画出内裤 / 短裤线稿，并在图例中明确写：

- `талия трусиков`
- `талия шорт`
- `длина шорт`

### 肩带字段

如果显示肩带长：

- 图例写 `длина бретели`
- 测量线画在肩带上
- 不要画在衣身侧边

### 袖长字段

如果有袖长：

- 必须使用 T3A 或带袖线稿
- 图例写 `длина рукава`
- 测量线画在袖子上

### 长度字段

如果原字段写“不含肩带”，俄语必须写：

`длина без бретелей`

如果原字段写“含肩带 / 肩顶至下摆”，俄语必须写：

`длина с бретелями`

如果原字段没有明确含不含肩带，可写：

`длина сорочки` 或 `длина изделия`

## 底部提示

底部只保留两句。

第一句固定：

`Если вы между двумя размерами — выбирайте больший.`

第二句必须根据最终显示字段动态生成，不要提到图中没有显示的字段。

建议：

- 字段包含袖长：`Для комплекта важны длина рукава, талия шорт и бёдра. Допуск ±1–3 см.`
- T1 连体衣：`Для боди ориентируйтесь на рост и длину изделия. Допуск ±1–3 см.`
- T2 长裤睡衣：`Для комплекта важны талия, бёдра и длина брюк. Допуск ±1–3 см.`
- T3 短裤睡衣：`Для комплекта важны талия шорт, бёдра и посадка верха. Допуск ±1–3 см.`
- T4/T5 且显示 `грудь`、`талия`、`длина`：`Для сорочки важны грудь, талия и длина. Допуск ±1–3 см.`
- 只显示 `под грудью`、长度、肩带长：`Для этой модели важны длина изделия, обхват под грудью и длина бретели. Допуск ±1–3 см.`
- T6 长款：`Для длинной сорочки важны рост и длина изделия. Допуск ±1–3 см.`

## 图例排版

- 1–4 个字段：可以单列。
- 5–6 个字段：优先两列，避免拥挤。
- 图例必须与测量线、右下表完全一致。
- 图例 A/B/C/D/E/F 顺序必须与右下表行顺序一致。

## 生成前自检

每张图生成前必须检查：

1. 该货号是否属于第一批。
2. 是否读取到了推荐身高。
3. 是否读取到了推荐体重。
4. 是否读取到了原始尺寸数据。
5. 尺码列是否只包含当前货号实际尺码。
6. 最终显示字段是否都有原始数据来源。
7. 是否存在整行 `—`，如有则删除该字段。
8. 图例、测量线、右下表行是否完全一致。
9. 底部提示是否只提到最终显示字段。
10. 是否错误展示了内裤腰围 / 裤腰围在裙身腰部。
11. 是否错误把多个货号的数据合并成范围。
12. 是否错误使用了模板图里的示例数字。

每生成一张图片，同时输出 CSV 或 Excel 检查表，字段包括：

- 货号
- SKU
- 使用模板
- 图片标题
- 实际尺码列
- 最终显示字段
- 删除的模板字段

- 是否存在整行 —
- 是否存在未匹配字段
- 右下表数据来源是否完整
- 底部提示
- 是否成功生成
- 图片文件名
- 备注

示例：

- 货号：5346
- SKU：FZNY5346BK
- 使用模板：T1
- 图片标题：Боди / комбинезон
- 实际尺码列：S/M/L/XL/XXL

- 是否成功生成：是
- 备注：原表无臀围，已删除 `бёдра`，改用肩带长

## 文件命名

每个货号生成一张 PNG。

格式：

`货号_SKU_模板编号_俄语类别_尺码图.png`

示例：

- `5346_FZNY5346BK_T1_bodi_尺码图.png`
- `7889_FZNY7889WE_T3A_shorty_rukav_尺码图.png`
- `30369_FZNY30369BK_T5_sorochka_尺码图.png`

## 问题处理

1. 完全缺尺码数据：不生成图片，记录到检查表。
2. 只有 S 码：只显示 S 列，不编造 M/L/XL。
3. 缺少某个部位：删除该字段，不显示整行 `—`。
4. 字段语义不确定：保留更核心字段，删除可疑字段。
5. 下装字段无法正确画线：删除字段，不要画在裙身上。
6. 推荐身高体重缺失：可以只生成 `Замеры изделия` 表，但检查表中标记推荐参数缺失。

## 最终质量标准

每张图必须满足：

1. 右上表是横排尺码。
2. 右下表是横排尺码。
3. A/B/C/D/E/F 是纵向测量项目。
4. 图例 = 测量线 = 右下表行 = 底部提示。
5. 没有整行 `—`。
6. 没有原表不存在的部位。
7. 没有把多个货号合并成范围。
8. 没有把内裤/裤子尺寸画在裙身上。
9. 没有模板图里的旧示例数字。
10. 图片简洁，不密集，适合 Wildberries / Ozon 商品附图。

## 执行方式

不要在旧 PNG 上覆盖文字或局部修补。

请基于最新版本的生成代码 / 最新模板逻辑进行修改，然后重新渲染输出新的 PNG。

保留当前版本中已经满意的视觉风格：

- 白底
- 浅粉色标题栏
- 黑色俄语文字
- 左侧线稿
- 右侧双表格
- 底部提示框

但必须重构：

- 数据逻辑
- 字段逻辑
- 横排尺码表布局
- 动态底部提示逻辑

## 样图优先

不要先跑完整批。
